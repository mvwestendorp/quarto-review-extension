# Code Analysis Documents

This directory contains code quality analysis, test reports, technical limitations, and improvement tracking for the Quarto Review Extension.

## Contents

- [Known Limitations](./KNOWN_LIMITATIONS.md) - Current technical limitations and constraints
- [Test Suite Status](./TEST_SUITE_STATUS.md) - Test coverage analysis and skipped tests
- [Improvements Summary](./IMPROVEMENTS_SUMMARY.md) - Test suite improvements and enhancements

## Purpose

These documents help developers understand:
- Current code quality metrics
- Areas needing improvement
- Test coverage gaps
- Technical debt
- Performance bottlenecks

## How to Use

### For New Contributors
Start with [Known Limitations](./KNOWN_LIMITATIONS.md) to understand current constraints before diving into development.

### For Code Reviews
Reference these analysis documents when reviewing PRs to ensure improvements align with identified issues.

### For Planning
Use these documents alongside [Planning docs](../planning/) to prioritize refactoring and improvement work.
